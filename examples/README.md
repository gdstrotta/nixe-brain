# Domain Examples

These examples show how to calibrate the Nixe Brain pattern for specific domains.
Each example provides:

- **project-instructions.md** — A complete, ready-to-use Claude project instructions
  template calibrated for that domain, with placeholder values specific to the domain
- **suggested-sources.md** — A prioritized list of what to ingest for that domain

## Available Examples

| Domain | Typical users | Key characteristics |
|---|---|---|
| [Legal](legal/) | Law firms, legal professionals | Legislation, case law, procedures |
| [Startup](startup/) | Founders, startup teams | Clients, strategy, product, operations |
| [Research](research/) | Academics, independent researchers | Papers, notes, methodology |

## How to Use an Example

1. Open the domain folder that best matches your use case
2. Read `project-instructions.md` — this is your starting template
3. Replace all `{{PLACEHOLDER}}` values with your specific information
4. Follow `suggested-sources.md` for your first wave of ingestions

## Adapting to Your Domain

If none of the examples match your domain exactly:

1. Start with the [generic single-user template](../templates/project-instructions/single-user.md)
2. Adapt the **Tags** section to your domain vocabulary
3. Adapt the **Naming Conventions** to match your domain's standard naming
4. Use the **suggested-sources** format from the closest example as a starting point
